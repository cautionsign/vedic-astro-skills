# 宫位诊断框架

> 用途：core在Step 3宫位诊断时参考
> 每个宫位执行：管理者审计 + 租客审计 + 硬件带宽 + 分盘交叉（如有）

---

## 诊断三维度

### 1. 管理者（宫头星/宫主）
```
查看宫主的：
  P1身份 → 它是什么角色？
  去向(落宫) → 它去了哪个宫位？把这个宫的能量带到了哪里？
  P7尊贵度 → 它健不健康？
  P9 Shadbala → 它有没有力量执行？
```

### 2. 租客（宫内行星）
```
查看宫内星的：
  自带属性 → 吉星进驻=加持，凶星进驻=干扰
  P10相位 → 有没有其他星注视这个宫位？
  合相情况 → 多颗星同宫=事务繁忙
```

### 3. 硬件带宽（SAV）
```
SAV > 32 = 自动化运转
SAV 26-32 = 标准配置
SAV 20-25 = 高阻力
SAV < 20 = 漏雨（需要大量个人努力）
```

## 宫位评级

```
创始人模式：宫主强+宫内有吉星+SAV高 → 自主掌控
经理人模式：宫主中等+SAV中等 → 正常运作
吉祥物模式：宫主弱+有吉星相位 → 有贵人帮但自己弱
飘萍模式：宫主弱+无吉星+SAV低 → 该领域缺乏根基
```

## 事件关联陈述模板

在分析重点宫位时，自然嵌入Dasha事件关联：

```
"你的[X]宫主是[行星]，力量指数[Y]%([强/中/弱])，
 落在[Z]宫（[领域]）。

 在[大运行星]大运期间（[年份]-[年份]），
 你的[X宫领域]应该经历了[描述]...

 [如果宫主强]：这段时期你在这个领域有明确的进展和掌控感。
 [如果宫主弱]：这段时期这个领域可能给你带来了压力或不确定感。"
```

> 注意：这是陈述不是提问。用户不需要回答。
> 目的：展示分析实力 + 让用户在阅读中自行验证。

## Badhaka（障碍星）审计

宫位诊断完成后，检查Badhaka宫和Badhakesh（障碍星）：

```
Lagna类型 → Badhaka宫：
  活动星座(Aries/Cancer/Libra/Capricorn) → 11宫
  固定星座(Taurus/Leo/Scorpio/Aquarius) → 9宫
  变动星座(Gemini/Virgo/Sagittarius/Pisces) → 7宫

Badhakesh = Badhaka宫的宫主
  → Badhakesh落入的宫位 = 容易遭遇"莫名障碍"的领域
  → Badhakesh与特定宫位形成关联(合相/相位) → "好事多磨"

在宫位诊断报告中，标注Badhakesh影响的宫位事务。
不过度渲染，仅作为"阻力来源标记"使用。
```

```

  → Maraka行星大运期 = 健康风险窗口（不是"必然出事"）

注意：Maraka标签不用于日常分析，仅在用户问健康问题时引用。
```
