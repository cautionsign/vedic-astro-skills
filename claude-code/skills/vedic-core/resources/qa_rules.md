# Q&A模式规则

> 用途：core在对话中已有完整报告，或用户附带报告文件时，不重跑pipeline，进入答疑
> 适用范围：不限于技术追问，包括任何人生问题

---

## 处理规则

1. **时机类问题**：用户问"什么时候找到工作/对象/买房？"
   → 查Dasha时间线 + 对应宫位（10宫/7宫/4宫）+ SAV
   → 找到激活期："[行星]大运的[行星]小运期间（[年月]-[年月]），你的[X]宫被激活..."
   → 给具体时间窗口，不说"总有一天"

2. **方向类问题**：用户问"适合做什么行业/学什么/去哪个城市？"
   → 引用10宫+AmK+格局（行业） / 4/5宫（学业） / 4/9/12宫（方位）
   → 给具体建议

3. **运势类问题**：用户问"今年/明年运势怎么样？"
   → 查当前Dasha+Antardasha → 对应宫主管什么 → 综合判断
   → 按领域分述（事业/财运/感情/健康）

4. **技术追问**：用户问"8宫怎么理解""Moon为什么弱"
   → 引用报告中的P1-P12数据+宫位诊断，展开解释

5. **化解类问题**：用户问"怎么改善/怎么补救？"
   → 基于弱点宫位给行动建议（不推荐宝石/仪式等迷信手段）
   → 方向：利用强星赛道、避开弱星领域、时机选择

6. **偏差反馈**：用户说"不像我""分析不准"
   → 提示："如果多项分析都感觉偏差，可能是出生时间的问题。可以说'校准时间'运行vedic-rectifier。"

---

## Q&A约束

- **正反双审（最重要）**：回答任何判断性问题时，必须同时列出支持和反对的数据，不准只挑一边。格式：
  ```
  支持[X]的数据：
    - [数据1] → [含义]
    - [数据2] → [含义]
  
  制约[X]的数据：
    - [数据3] → [含义]
    - [数据4] → [含义]
  
  综合判断：[基于以上正反数据的结论]
  ```
  **禁止行为**：
  - 用户想听好话 → 只列支持数据 → ❌ 卖乐观
  - 用户有焦虑 → 只列制约数据 → ❌ 卖悲观
  - 忽略BAV/SAV/Shadbala中与用户预期矛盾的数据点 → ❌ 选择性引用

- **基于数据**：所有回答引用structured_data和已有报告，不编造
- **引用原文**：关键结论必须附上报告原文出处（如"p4第290行"），方便用户对照
- **区分层次**：结构层（有没有这个信号）和执行层（做起来顺不顺）分开讲，不混为一谈
- **语言风格**：70%大白话 + 20%数据 + 10%注释
- **文件保存**：每次Q&A保存为 `qa_主题.md`（如`qa_job_timing.md`），存入工作目录
- **跨领域**：如果问题属于career/love深度领域，引导用户跑对应skill获得更详细分析
